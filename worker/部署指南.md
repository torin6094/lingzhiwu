# Cloudflare Worker 部署指南

## 方案二：Worker 中转方案

这个 Worker 作为中间层，用服务端 API 访问 Cloudinary，解决前端直接访问的权限问题。

## 部署步骤

### 方法1：通过 Cloudflare Dashboard（推荐，无需命令行）

1. **登录 Cloudflare Dashboard**
   - 访问 https://dash.cloudflare.com
   - 使用你的 Cloudflare 账号登录

2. **创建 Worker**
   - 左侧菜单点击 "Workers & Pages"
   - 点击 "Create application"
   - 选择 "Create Worker"
   - 输入 Worker 名称，例如：`lingzhiwu-api`
   - 点击 "Deploy"

3. **编辑代码**
   - 部署后点击 "Edit code"
   - 删除默认代码，粘贴 `portfolio-api.js` 的内容
   - 点击 "Save and deploy"

4. **获取 Worker URL**
   - 部署成功后，页面会显示 Worker 地址
   - 类似：`https://lingzhiwu-api.yourname.workers.dev`
   - 复制这个地址

5. **测试 API**
   - 在浏览器中访问 Worker URL
   - 应该返回 JSON 格式的图片列表

### 方法2：使用 Wrangler CLI（需要命令行）

```bash
# 安装 wrangler
npm install -g wrangler

# 登录 Cloudflare
npx wrangler login

# 创建 Worker
npx wrangler init lingzhiwu-api

# 编辑 wrangler.toml 和 src/index.js
# 然后部署
npx wrangler deploy
```

## 后续配置

部署成功后，将 Worker URL 告诉 AI，更新前端代码即可。

## 工作原理

```
用户上传图片到 Cloudinary
        ↓
用户打开网页
        ↓
前端调用 Worker API (https://lingzhiwu-api.xxx.workers.dev)
        ↓
Worker 用 API Key 访问 Cloudinary
        ↓
返回图片列表给前端
        ↓
网页自动显示所有图片
```

## 优点

- 真正的实时更新
- 无需重新部署网站
- 用户只需上传图片到 Cloudinary 即可
